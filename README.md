# CS250-Software-Development-Lifecycle

## How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

I try to interpret user stories as well as I can as the user themselves. This helps me to understand the needs and requirements of the user story so I can better design the program. Without user stories, it makes it far more difficult to develop programs as the requirements can be very unclear. Implementing the user stories then makes it easier with this understanding as I now can internalize the request and solution. User stories help to take the ague requirements into clear, user‑focused goals.

## How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I generally like to start projects with either a roadmap or a design document which helps to organize my thoughts and ideas. With this document, I can then start tackling ideas like organizing OOP concepts in the program and building test suites into the program to check for. Lastly, I prefer to more iteratively developing the code to verify my ideas as I go. This is like working in sprints as you aim to complete smaller tasks that build up to a larger goal.

## What does it mean to be a good team member in software development?

To be a strong team member, it means to contribute more than technical skill. Effective collaboration means communicating clearly, supporting teammates, and being open to feedback. Good team members help create an environment where ideas can be shared freely, challenges are welcomed, and the team can deliver high‑quality work together. In Agile environments, this collaborative mindset is essential for success and helps to drive high performing teams.
